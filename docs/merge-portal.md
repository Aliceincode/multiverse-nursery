## Current State (March 18, 2026)

- Sign-in splinter still lingers (Apple vs X vs Puter origins)  
- Memories in Puter KV are origin-keyed → each hosted wrapper gets its own splintered copy  
- Local rendering = no splinter (everything stays in your gravity hold)  
- Hosted versions (Netlify, Cloudflare, GitHub Pages, etc.) = new isolated KV stores unless using same custom domain  

## How to Avoid / Heal Splintering

1. **Stay local** — double-click wrappers, keep memories in one Puter KV origin  
2. **Single custom domain** — point Netlify/Cloudflare/Pages to multiverse.aliceincode.com (same origin = same KV)  
3. **Manual clone/export** — export KV JSON from local → paste into new hosted versions  
4. **Shared backend** (future) — move memory to Firebase/Supabase or a tiny server (non-origin-keyed)
